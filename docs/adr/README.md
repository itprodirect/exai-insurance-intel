# ADR Index

Architecture Decision Records capture durable decisions that should outlive individual sessions or issue comments.

## Numbering

- Use zero-padded sequential identifiers: `ADR-0001`, `ADR-0002`, and so on.
- Never renumber old ADRs.
- If a decision is replaced, add a new ADR that supersedes the old one.

## Format

Each ADR should include:

- title
- date
- status
- context
- decision
- consequences
- related roadmap items or issues
